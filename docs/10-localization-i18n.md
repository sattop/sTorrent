# sTorent: локализация и смена языка

## Цель

`sTorent` должен поддерживать смену языка интерфейса в настройках. Пользователь должен иметь возможность выбрать язык без переустановки приложения.

## Стартовые языки

Для MVP или первой стабильной версии подготовить 4 языка:

- Русский (`ru`)
- English (`en`)
- Español (`es`)
- 中文 (`zh`)

Если полная локализация всех 4 языков не успевает в MVP, минимально обязательны `ru` и `en`, а `es` и `zh` добавляются в следующей версии.

## Поведение в интерфейсе

Настройка языка находится в:

```text
Настройки -> Интерфейс -> Язык
```

Требования:

- текущий язык выбран в выпадающем списке;
- после смены языка интерфейс обновляется сразу, если это технически возможно;
- если мгновенное обновление сложно, приложение показывает предложение перезапустить UI;
- выбранный язык сохраняется между запусками;
- системный язык используется по умолчанию, если он поддерживается;
- если системный язык не поддерживается, используется English.

## Что должно переводиться

- навигация;
- кнопки;
- заголовки;
- статусы торрентов;
- ошибки;
- уведомления;
- настройки;
- подсказки;
- диалог добавления torrent/magnet;
- WebUI, если он включен;
- релизные страницы и README по возможности.

## Что не переводить автоматически

- имена файлов;
- названия торрентов;
- tracker URLs;
- magnet-ссылки;
- системные пути;
- технические ключи настроек;
- логи в формате, который нужен для диагностики.

## Технические требования

- Все строки интерфейса должны храниться в словарях локализации.
- В коде не должно быть захардкоженных пользовательских строк.
- Ключи переводов должны быть стабильными.
- Для множественных чисел использовать plural rules.
- Форматы даты, времени и чисел должны зависеть от выбранной локали.
- Направление текста должно быть готово к расширению для RTL-языков в будущем, но RTL не обязателен для MVP.

## Пример структуры файлов

```text
src/
  i18n/
    ru.json
    en.json
    es.json
    zh.json
```

Пример ключей:

```json
{
  "nav.downloads": "Загрузки",
  "nav.settings": "Настройки",
  "torrent.status.downloading": "Скачивается",
  "torrent.status.seeding": "Раздается",
  "settings.interface.language": "Язык"
}
```

## Проверка качества

Перед релизом AI или разработчик должен проверить:

- все ключи есть во всех языках;
- нет пустых переводов;
- длинный текст не ломает кнопки и панели;
- язык можно переключить и сохранить;
- после перезапуска выбранный язык остается активным.

## Правило для AI-агента

При добавлении новой UI-строки AI должен:

1. добавить ключ в базовую локаль;
2. добавить переводы во все поддерживаемые языки;
3. проверить, что в коде используется ключ, а не обычная строка;
4. не смешивать русский и английский текст в одном экране, если это не имя продукта или технический термин.
