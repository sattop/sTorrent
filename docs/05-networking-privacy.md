# sTorent: сеть, приватность и устойчивость соединения

## Цель

Сетевые настройки `sTorent` должны помогать пользователю получать стабильную работу BitTorrent в разных сетях, не нарушая правила сторонних сервисов и не подменяя приложение под другой бренд или протокол.

## Разрешенные функции

### BitTorrent protocol encryption

Добавить настройку шифрования BitTorrent-соединений:

- `Отключено`
- `Разрешено`
- `Предпочитать шифрование`
- `Требовать шифрование`

Интерфейс должен честно объяснять, что это не гарантирует обход сетевых ограничений, а только включает штатные возможности протокола.

### Прокси

Поддержать настройки прокси, если выбранная torrent-библиотека это позволяет:

- SOCKS5;
- HTTP proxy;
- прокси только для трекеров;
- прокси для peer-соединений;
- авторизация по логину и паролю;
- тест подключения.

Пароль прокси должен храниться безопасно: через системное хранилище секретов, если оно доступно.

### VPN-friendly режим

Добавить режим для пользователей, которые сами используют VPN:

- выбор сетевого интерфейса;
- аварийная остановка загрузок при исчезновении выбранного интерфейса;
- запрет соединений вне выбранного интерфейса;
- предупреждение, если интерфейс недоступен.

Это не встроенный обход ограничений, а защита от случайного выхода трафика через неправильный интерфейс.

### Лимиты и расписания

Добавить:

- глобальный лимит загрузки;
- глобальный лимит отдачи;
- лимит на конкретный торрент;
- расписание дневных/ночных лимитов;
- быстрые профили `Без лимита`, `Экономия`, `Ночь`, `Ручной`.

### Частные трекеры

Для приватных трекеров:

- автоматически уважать private flag из `.torrent`;
- отключать DHT/PEX/LSD для private-раздач;
- показывать предупреждение перед изменением этих параметров.

## Недопустимые функции

В `sTorent` не нужно проектировать и реализовывать:

- маскировку трафика под Steam, Discord, YouTube, Telegram или другие сервисы;
- выбор чужого приложения как профиля имперсонации;
- подделку TLS/HTTP/QUIC-отпечатков сторонних сервисов;
- обход белых списков оператора через имитацию разрешенного сервиса;
- скрытие назначения трафика от сети через ложную идентификацию.

Такие функции создают риски для пользователя, нарушают правила сетей и могут привести к блокировкам, ошибочной маршрутизации, юридическим проблемам и недоверию к приложению.

## Безопасная альтернатива разделу "маскировка"

Вместо раздела `Маскировка трафика` создать раздел `Сетевой профиль`.

Профили:

- `Стандартный` - обычные настройки BitTorrent.
- `Приватный трекер` - DHT/PEX/LSD отключены, включена строгая работа с трекерами.
- `Шифрование` - предпочтение зашифрованным BitTorrent-соединениям.
- `Прокси` - весь поддерживаемый трафик идет через настроенный прокси.
- `VPN-интерфейс` - соединения разрешены только через выбранный сетевой интерфейс.
- `Экономия трафика` - ограничены скорость, подключения и активные загрузки.

## Диагностика сети

Добавить экран диагностики:

- проверка входящего порта;
- проверка доступности трекера;
- проверка DHT;
- проверка прокси;
- проверка выбранного сетевого интерфейса;
- отображение текущего внешнего IP, если пользователь явно запускает проверку;
- экспорт диагностического отчета без приватных данных.

## Текст для интерфейса

Пример предупреждения:

> sTorent не подменяет трафик под чужие сервисы. Для стабильной работы используйте шифрование BitTorrent, прокси, VPN-интерфейс, лимиты скорости и настройки трекеров.
