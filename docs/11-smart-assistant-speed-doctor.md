# sTorent: Smart Download Assistant и Speed Doctor

## Цель

Главная killer feature `sTorent` - умный помощник загрузки и диагностики скорости. Большинство torrent-клиентов показывают много чисел, но не объясняют, что происходит и что делать. `sTorent` должен помогать пользователю принимать правильные решения до начала загрузки и во время проблем со скоростью.

Фича состоит из двух частей:

- `Smart Download Assistant` - помогает выбрать оптимальные настройки при добавлении торрента.
- `Speed Doctor` - объясняет, почему загрузка идет медленно, и предлагает безопасные действия.

## Принципы

- Помощник не должен скрывать сложность ложными обещаниями.
- Любая рекомендация должна объяснять причину.
- Опытный пользователь может отключить помощника и настроить все вручную.
- Помощник не должен подменять трафик под чужие сервисы или предлагать обход сетевых правил.
- Все действия, которые меняют настройки сети, файлов или удаления данных, требуют явного подтверждения.

## Smart Download Assistant

### Когда показывается

Помощник открывается при:

- добавлении `.torrent`;
- добавлении magnet-ссылки после получения metadata;
- перетаскивании torrent-файла в окно;
- добавлении из watch folder, если для папки включено `Спрашивать настройки`;
- ручном выборе действия `Настроить умно`.

### Что анализирует

Перед стартом загрузки помощник анализирует:

- общий размер раздачи;
- список файлов;
- типы файлов;
- доступное место на диске;
- выбранную папку сохранения;
- существующие файлы с похожими именами;
- категории и теги пользователя;
- активные загрузки;
- текущие лимиты скорости;
- состояние выбранного сетевого профиля;
- private flag в `.torrent`;
- количество сидов/пиров, если данные уже доступны;
- настройки DHT/PEX/LSD;
- доступность трекеров, если проверка возможна быстро.

### Что предлагает

Помощник может предложить:

- папку сохранения;
- категорию;
- теги;
- выбрать только нужные файлы;
- отключить лишние файлы;
- режим загрузки;
- лимит скорости;
- правило сидирования;
- включить проверку данных после завершения;
- запустить сразу или добавить на паузе;
- предупредить о нехватке места;
- предупредить о конфликте имен;
- предупредить, что раздача приватная и DHT/PEX/LSD должны быть отключены.

### Профили загрузки

#### Максимальная скорость

Назначение:

- скачать как можно быстрее.

Поведение:

- нет искусственных лимитов скорости;
- обычный rarest-first режим;
- не включать последовательную загрузку по умолчанию;
- использовать DHT/PEX/LSD, если torrent не приватный;
- разрешить больше peer-соединений в рамках безопасных лимитов.

Когда предлагать:

- пользователь скачивает большой файл;
- нет активных ограничений;
- диск и сеть не перегружены.

#### Смотреть во время загрузки

Назначение:

- начать смотреть медиафайл до полного завершения загрузки.

Поведение:

- включить последовательную загрузку, если core поддерживает;
- повысить приоритет первого медиафайла;
- повысить приоритет первых и последних частей файла, если core поддерживает;
- не применять глобально ко всем торрентам;
- показать предупреждение, что режим может снизить эффективность swarm.

Когда предлагать:

- в раздаче есть видео или аудио;
- выбран один основной медиафайл;
- у раздачи достаточно сидов.

#### Ночной режим

Назначение:

- скачивать активнее ночью и меньше мешать днем.

Поведение:

- включить расписание лимитов;
- днем использовать спокойные лимиты;
- ночью использовать повышенные лимиты;
- не менять системные сетевые настройки.

Когда предлагать:

- пользователь уже использует расписание;
- активных загрузок много;
- пользователь выбирает большой torrent.

#### Приватный трекер

Назначение:

- корректная работа с private-раздачами.

Поведение:

- уважать private flag;
- отключить DHT/PEX/LSD для private torrent;
- применить ratio/time сидирования;
- не добавлять внешние трекеры;
- не публиковать лишнюю информацию в логах.

Когда предлагать:

- torrent содержит private flag;
- трекер похож на private tracker;
- пользователь вручную выбирает этот профиль.

#### VPN-интерфейс

Назначение:

- разрешить соединения только через выбранный сетевой интерфейс.

Поведение:

- привязать соединения к выбранному интерфейсу, если core это поддерживает;
- остановить загрузки при исчезновении интерфейса;
- показать предупреждение, если интерфейс недоступен;
- не обещать анонимность и не заменять VPN.

Когда предлагать:

- пользователь ранее выбрал VPN-friendly режим;
- выбранный интерфейс активен;
- пользователь скачивает через сеть, где важен контроль интерфейса.

#### Экономия трафика

Назначение:

- снизить сетевую нагрузку.

Поведение:

- ограничить скорость;
- ограничить количество соединений;
- ограничить количество активных загрузок;
- отключить автозагрузку RSS, если пользователь подтвердит;
- не менять текущие раздачи без подтверждения.

Когда предлагать:

- пользователь на медленной сети;
- включен лимитный тариф;
- пользователь вручную выбирает профиль.

#### Ручной режим

Назначение:

- ничего не применять автоматически.

Поведение:

- показать все настройки;
- не менять значения без выбора пользователя;
- сохранить выбор пользователя как шаблон, если он согласится.

## Speed Doctor

### Где находится

Speed Doctor должен быть доступен:

- на главном экране рядом с общей скоростью;
- в карточке каждого торрента;
- в контекстном меню торрента;
- в разделе `Сеть -> Диагностика`;
- в уведомлении, если скорость долго ниже ожидаемой.

### Когда запускать автоматически

Автоматическая подсказка разрешена, если:

- загрузка активна больше 3 минут;
- скорость ниже 10% от ожидаемой или ниже пользовательского порога;
- torrent не стоит на паузе;
- нет явного глобального лимита, объясняющего скорость.

Автоматическая подсказка не должна спамить. Для одного торрента показывать ее не чаще одного раза за 30 минут.

### Что проверяет

Speed Doctor проверяет:

- torrent стоит на паузе или в очереди;
- включен глобальный лимит скорости;
- включен лимит на torrent;
- включен лимит категории или тега;
- активен ночной/экономный профиль;
- мало сидов;
- нет подключенных пиров;
- трекер не отвечает;
- DHT выключен для публичной раздачи;
- DHT/PEX/LSD правильно выключены для private-раздачи;
- входящий порт закрыт;
- UPnP/NAT-PMP выключен или не сработал;
- выбранный сетевой интерфейс недоступен;
- прокси не подключается;
- диск перегружен;
- нет места на диске;
- файл заблокирован другим приложением;
- очередь загрузок занята другими торрентами;
- выбранные файлы имеют низкий приоритет;
- торрент почти мертвый: нет сидов долгое время.

### Формат результата

Результат должен быть понятным:

```text
Скорость низкая, потому что:
1. У торрента только 1 сид.
2. Входящий порт закрыт.
3. Включен дневной лимит 500 КБ/с.

Что можно сделать:
- Проверить порт.
- Включить UPnP/NAT-PMP.
- Временно переключиться на профиль "Максимальная скорость".
```

Для UI лучше показывать:

- главный диагноз;
- 2-4 причины по важности;
- кнопки действий;
- ссылку `Показать подробности`;
- технический отчет для копирования.

### Приоритет причин

Причины сортируются по ожидаемому влиянию:

1. Пользовательские лимиты и пауза.
2. Нехватка сидов/пиров.
3. Трекер/DHT/PEX/LSD.
4. Порт/NAT/UPnP.
5. Прокси/VPN-интерфейс.
6. Диск и файловая система.
7. Очередь и приоритеты.
8. Прочие ошибки.

### Действия из Speed Doctor

Допустимые быстрые действия:

- снять временный лимит;
- переключить профиль загрузки;
- включить/выключить DHT для публичного torrent;
- проверить порт;
- включить UPnP/NAT-PMP;
- выбрать другой сетевой интерфейс;
- проверить прокси;
- поставить torrent выше в очереди;
- повысить приоритет файлов;
- открыть папку;
- запустить проверку данных;
- показать трекеры;
- скопировать диагностический отчет.

Недопустимые действия:

- подменять трафик под Steam, Discord или другой сервис;
- обходить белые списки оператора через имперсонацию;
- автоматически удалять данные;
- автоматически отключать private-mode;
- отправлять диагностический отчет без подтверждения пользователя.

## UX

### Главный экран

Добавить кнопку:

```text
Почему медленно?
```

или компактный вариант:

```text
Speed Doctor
```

Если проблем нет:

```text
Сеть выглядит нормально. Ограничений не найдено.
```

Если проблема найдена:

```text
Найдено 3 причины низкой скорости
```

### Диалог добавления торрента

Добавить блок:

```text
Умная настройка
Профиль: Максимальная скорость
Причина: это большой публичный torrent, сейчас нет активных лимитов, места на диске достаточно.
```

Пользователь может:

- принять профиль;
- выбрать другой профиль;
- раскрыть детали;
- перейти в ручной режим.

### Карточка торрента

Показывать небольшой статус:

- `Оптимально`
- `Ограничено лимитом`
- `Мало сидов`
- `Проблема с портом`
- `Трекер не отвечает`
- `Диск занят`

Статус не должен занимать много места и не должен мешать основным действиям.

## Архитектура

### Smart Assistant Engine

Отвечает за:

- анализ metadata;
- выбор профиля;
- предложения папки, категории и тегов;
- проверку конфликтов;
- генерацию объяснения;
- применение выбранного профиля.

Входные данные:

- torrent metadata;
- app settings;
- category/tag rules;
- disk status;
- network status;
- user history;
- active torrents.

Выходные данные:

- рекомендуемый профиль;
- список предложений;
- список предупреждений;
- список действий, требующих подтверждения.

### Speed Doctor Engine

Отвечает за:

- сбор диагностических сигналов;
- расчет причин низкой скорости;
- сортировку причин;
- генерацию понятного объяснения;
- подготовку быстрых действий;
- экспорт отчета.

Входные данные:

- torrent runtime stats;
- tracker status;
- peer stats;
- DHT/PEX/LSD status;
- limits;
- queue status;
- port status;
- proxy status;
- network interface status;
- disk status;
- recent errors.

Выходные данные:

- health status;
- primary reason;
- ranked reasons;
- recommended actions;
- technical details.

## Модель данных

Пример результата Smart Assistant:

```json
{
  "profile": "max_speed",
  "confidence": 0.82,
  "reasons": [
    "Большой публичный torrent",
    "Нет активных лимитов",
    "Достаточно места на диске"
  ],
  "suggestions": [
    {
      "type": "category",
      "value": "Downloads",
      "requiresConfirmation": false
    }
  ],
  "warnings": []
}
```

Пример результата Speed Doctor:

```json
{
  "status": "warning",
  "primaryReason": "incoming_port_closed",
  "reasons": [
    {
      "code": "incoming_port_closed",
      "severity": "high",
      "message": "Входящий порт закрыт",
      "action": "check_port"
    },
    {
      "code": "low_seed_count",
      "severity": "medium",
      "message": "У торрента мало сидов",
      "action": "show_peers"
    }
  ],
  "actions": [
    "enable_upnp",
    "check_port",
    "copy_report"
  ]
}
```

## Реализация по этапам

### Этап A: ручные профили

- добавить профили;
- дать пользователю выбирать профиль при добавлении;
- применять настройки профиля;
- сохранять последний выбранный профиль.

### Этап B: простые рекомендации

- анализировать размер, тип файлов, private flag и свободное место;
- предлагать профиль;
- показывать объяснение;
- не менять настройки без подтверждения.

### Этап C: Speed Doctor v1

- проверять паузу, очередь, лимиты, сидов, трекер, порт и место на диске;
- показывать 2-4 причины;
- добавить быстрые действия;
- добавить копирование отчета.

### Этап D: расширенная диагностика

- проверка прокси;
- проверка VPN-интерфейса;
- проверка DHT/PEX/LSD;
- учет дисковой нагрузки;
- учет истории ошибок.

### Этап E: умные правила

- запоминать выбор пользователя;
- предлагать категории и теги;
- предлагать папку по типу файлов;
- поддержать шаблоны профилей.

## Acceptance criteria

Фича считается готовой для первой версии, если:

- пользователь может выбрать профиль при добавлении torrent/magnet;
- помощник объясняет, почему предлагает профиль;
- Speed Doctor запускается для отдельного торрента;
- Speed Doctor находит минимум: пауза, очередь, лимит, мало сидов, ошибка трекера, закрытый порт, нехватка места;
- каждое действие требует подтверждения, если меняет настройки;
- private-раздачи не получают небезопасных рекомендаций;
- диагностический отчет не содержит секреты, пароли прокси и лишние персональные данные.

## Инструкция для AI-агента

При реализации этой фичи AI должен:

1. Сначала сделать ручные профили без автоматического "магического" поведения.
2. Затем добавить рекомендации с объяснениями.
3. Затем добавить Speed Doctor v1.
4. Не пытаться сразу реализовать машинное обучение.
5. Использовать обычный rules engine: понятные правила, веса и причины.
6. Держать все тексты через i18n.
7. Не добавлять имперсонацию трафика или обход сетевых ограничений.
8. Покрыть правила unit-тестами.
