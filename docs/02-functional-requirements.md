# sTorent: функциональные требования

## MVP

### Управление торрентами

- Добавление `.torrent` файлов.
- Добавление magnet-ссылок.
- Пауза, продолжение и удаление загрузок.
- Выбор папки сохранения.
- Категории с отдельными папками сохранения.
- Теги для фильтров и правил.
- Выбор отдельных файлов внутри торрента.
- Отображение прогресса по каждому торренту.
- Отображение общей скорости загрузки и отдачи.
- Отображение сидов, пиров, ETA, размера и статуса.
- Очередь загрузок с приоритетами.

### Локализация

- Переключение языка интерфейса в настройках.
- Стартовые языки: Русский, English, Español, 中文.
- Выбор языка сохраняется между запусками.
- По умолчанию используется системный язык, если он поддерживается.
- Все пользовательские строки должны быть вынесены в словари локализации.

### Состояния торрента

- `Добавляется`
- `Проверяется`
- `В очереди`
- `Скачивается`
- `На паузе`
- `Раздается`
- `Завершен`
- `Ошибка`

### Работа с файлами

- Открыть папку загрузки.
- Открыть скачанный файл системным приложением.
- Проверить целостность данных.
- Переместить данные в другую папку.
- Удалить только торрент из списка.
- Удалить торрент вместе с данными после явного подтверждения.

### Сетевые функции

- DHT.
- PEX.
- LSD для локальной сети.
- Поддержка публичных и частных трекеров.
- Настраиваемые лимиты скорости загрузки и отдачи.
- Ограничение количества активных загрузок.
- Ограничение количества подключений.
- Выбор входящего порта.
- Проверка доступности входящего порта.

### Автоматизация

- Watch folders для автоматического добавления `.torrent` файлов.
- Избранные папки сохранения в диалоге добавления.
- Правила сидирования по ratio и времени.
- Очередь проверки данных.
- После MVP: RSS-ленты и правила автозагрузки.

### Smart Download Assistant и Speed Doctor

- Умные профили загрузки: `Максимальная скорость`, `Смотреть во время загрузки`, `Ночной режим`, `Приватный трекер`, `VPN-интерфейс`, `Экономия трафика`, `Ручной режим`.
- При добавлении torrent/magnet помощник предлагает профиль, папку, категорию, теги, выбор файлов и правила сидирования.
- Каждая рекомендация должна иметь понятное объяснение.
- Пользователь может принять рекомендацию, изменить ее или перейти в ручной режим.
- Speed Doctor должен объяснять низкую скорость по конкретным причинам: лимиты, очередь, мало сидов, трекер, порт, DHT/PEX/LSD, прокси, сетевой интерфейс, диск, место на диске, приоритет файлов.
- Быстрые действия Speed Doctor не должны выполняться без подтверждения, если они меняют настройки.
- Speed Doctor не должен предлагать подмену трафика под чужие сервисы или обход сетевых правил.

### Приватность и безопасность

- Возможность отключить DHT, PEX и LSD для приватных раздач.
- Поддержка прокси SOCKS5/HTTP для трекеров и peer-соединений, если это поддерживает выбранная torrent-библиотека.
- Возможность привязать клиент к конкретному сетевому интерфейсу.
- Возможность отключить телеметрию полностью.
- Локальное хранение настроек.

## Не входит в проект

- Подмена трафика под Steam, Discord, браузер, мессенджер или другой сторонний сервис.
- Обход белых списков, DPI, блокировок или правил оператора через имперсонацию чужого приложения.
- Встроенные механизмы скрытого обхода сетевых ограничений.
- Автоматический поиск пиратского контента.
- Встроенный каталог нелегального контента.

## Нефункциональные требования

- Быстрый запуск: целевое время открытия приложения до 2 секунд на типичном desktop-устройстве.
- Стабильная работа с 100+ торрентами в списке.
- UI не должен зависать при активных загрузках.
- Все длительные операции выполняются в фоне.
- Ошибки должны показываться понятным языком.
- Логи должны помогать диагностировать сетевые и файловые проблемы.
- Windows-версия должна распространяться как `.exe` установщик.
- Релизные `.exe` сборки должны автоматически прикрепляться к GitHub Release.

## Платформы

Приоритет:

1. Windows 10/11.
2. Linux.
3. macOS.

Для MVP можно начать только с Windows, если это ускоряет разработку.

## Распространение

Основной формат для Windows:

- `sTorent-Setup.exe` - установщик приложения.
- `sTorent Portable.exe` - дополнительная portable-версия после MVP.

Публикация:

- исходный код хранится в GitHub-репозитории;
- сборки создаются через GitHub Actions;
- при создании git-тега `v*` Windows `.exe` автоматически загружается в GitHub Release;
- вместе с `.exe` публикуется checksum-файл для проверки целостности.
