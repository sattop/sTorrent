# sTorent: дорожная карта

## Принцип работы AI

Разработка должна идти поэтапно по документу [09-ai-step-by-step-workflow.md](09-ai-step-by-step-workflow.md). AI-агент сначала завершает текущий этап, запускает проверки и только потом переходит к следующему этапу.

## Этап 1: прототип интерфейса

Цель: показать внешний вид и основные сценарии без полноценного torrent-core.

Задачи:

- главный экран загрузок;
- экран добавления torrent/magnet;
- модальное окно выбора файлов;
- экран настроек;
- темная и светлая темы;
- переключение языка интерфейса;
- UI для профилей загрузки;
- UI для Speed Doctor;
- моковые данные для списка загрузок;
- базовые состояния: скачивается, пауза, завершено, ошибка.

## Этап 2: MVP torrent-core

Цель: получить рабочий клиент для реальных загрузок.

Задачи:

- добавление `.torrent`;
- добавление magnet-ссылок;
- получение metadata;
- загрузка файлов;
- пауза и продолжение;
- проверка хешей;
- категории и теги;
- словари локализации `ru`, `en`, `es`, `zh`;
- ручные профили загрузки;
- Smart Download Assistant v1;
- Speed Doctor v1;
- выбор файлов, папки, категории и тегов в диалоге добавления;
- отображение скорости и прогресса;
- простые лимиты скорости;
- сохранение состояния между запусками.

## Этап 3: настройки сети и приватности

Цель: дать пользователю контроль над сетевым поведением.

Задачи:

- DHT/PEX/LSD переключатели;
- private torrent режим;
- входящий порт;
- UPnP/NAT-PMP;
- выбор сетевого интерфейса;
- прокси;
- BitTorrent protocol encryption;
- диагностика сети.
- расширенная диагностика Speed Doctor;
- watch folders;
- избранные папки сохранения;
- правила сидирования.

## Этап 4: качество и удобство

Цель: довести приложение до комфортного ежедневного использования.

Задачи:

- системный трей;
- уведомления;
- расписания скорости;
- правила сидирования;
- поиск и фильтры;
- сортировка;
- категории;
- теги;
- RSS и правила автозагрузки;
- responsive WebUI или remote API;
- журнал ошибок;
- импорт настроек;
- экспорт настроек.

## Этап 5: релиз

Цель: подготовить стабильную версию.

Задачи:

- `.exe` установщик для Windows: `sTorent-Setup.exe`;
- GitHub Actions для автоматической сборки Windows `.exe`;
- автоматическая публикация `.exe` в GitHub Release при создании тега;
- checksum-файл для релизных сборок;
- автообновление;
- подпись приложения;
- документация пользователя;
- страница проекта;
- базовые тесты UI;
- тесты torrent-core;
- проверка производительности на большом списке торрентов.

## Критерии готовности MVP

- Пользователь может добавить `.torrent` и magnet-ссылку.
- Можно выбрать файлы и папку сохранения.
- Загрузка реально стартует и завершается.
- Прогресс, скорость, сиды, пиры и ETA обновляются в UI.
- Можно поставить загрузку на паузу и продолжить.
- После перезапуска приложение сохраняет список торрентов.
- Настройки скорости работают.
- Приложение не зависает при активной загрузке.
